Java关键字之static final finally

　　概括来说：在JAVA关键字中，final表示“无法改变的”，static表示“静态的”，即在类没有实例化的时候也能够通过类名进行访问，finally关键字表示在try catch之后进行一些收尾的工作。
　　下面依次对于三个关键字进行总结。

一、final关键字

  1. 当一个类被final修饰的时候，此类不能被继承，没有子类，final类中的方法默认都是final类型的。
  2. final方法不能被覆盖，能被继承。一个方法被修饰为final的好处在于1.方法锁定，不能被重写覆盖；2.高效，编译器在遇到final方式时会转入内嵌机制。
  3. final成员变量只能被赋值一次，而且在定义时就必须给定初值，哪怕是NULL，被赋值之后就不能够被改变。一种比较特殊的是“final空白”，即一个final变量在声明时没有赋值，但是这个变量在类进行初始化对象的时候进行赋值，即确保final空白在使用之前被赋值。
  4. final不能修饰构造方法。
  5. 因为父类的private方法不能够被子类方法覆盖，所以private方法默认是final类型的。
  6. 当函数形参被设置为final时，在函数内部变量不能够被修改。

二、static关键字

  1. static变量又叫做静态变量，与之对应的就是实例变量。静态变量在内存中只有一个拷贝，可以通过类名直接访问，也可以通过类对象来访问（不推荐这种方式）。对于实例变量，每实例化一个对象，就为实例变量分配一次内存，实例之间互不影响。
    ２．静态方法可以直接通过类名访问，静态方法中不能有this或者super关键字，只能访问类中的静态变量或者其他静态方法。因为静态方法独立于任何实例，所以必须被实现，所以一个静态方法不能是abstract类型的。
  2. static代码块，在类加载时候使用，可以有多个，执行顺序根据出现顺序而定。

三、static与final同时使用

  1. static与final同时修饰变量时，表示”全局变量”，不可修改，可以通过类名访问。其中比较特殊的是同时修饰一些容器类型的时候，容器本身不能够被改变，但是可以修改容器中存放的对象。
  2. static与final同时修饰方法时，表示不可覆盖，可以通过类名访问。

四、finally关键字

　　finally关键字是在try catch之后进行一些收尾工作，比如关闭链接或者释放资源。需要注意的一点是。finally代码快执行的时机实在try语句块中return语句执行之后，程序真正return之前。
